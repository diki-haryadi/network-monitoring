```bash

make

sudo ./network_monitor en0

# 3. Stop

```

---

## 🎮 Command Cheat Sheet

### Compile
```bash
make                    # Compile both programs
./build.sh              # Auto-install dependencies & build
clang++ ... -lpcap      # Manual compile
```

### Run
```bash
sudo ./network_monitor                    # Basic, choose interface
sudo ./network_monitor en0                # Basic, specific interface
sudo ./network_monitor_advanced           # Advanced, choose interface
sudo ./network_monitor_advanced en0       # Advanced, specific interface
```

### Analyze (Advanced)
```bash
cat traffic_report_*.json | jq '.'                          # Full report
cat traffic_report_*.json | jq '.domains | keys'            # List domains
cat traffic_report_*.json | jq '.packets[0:5]'              # First 5 packets
cat traffic_report_*.json | jq '.domains | to_entries | sort_by(.value.bytes) | reverse | .[0:10]'  # Top 10
```

### List Interfaces
```bash
ifconfig                # macOS/Linux
ip link show            # Linux
networksetup -listnetworkserviceorder  # macOS
```

## 📊 What Gets Captured

| Protocol | Port | Content |
|----------|------|---------|
| DNS | 53/UDP | Domain queries |
| HTTP | 80/TCP | Host header |
| HTTPS | 443/TCP | SNI/Host header |
| Custom | 8080/TCP | Host header |

## 📈 Statistics Explained

### Basic Monitor Shows
```
Domain                                  Packets         Bytes          
google.com                              245             156234         
```
- **Domain**: Website accessed
- **Packets**: How many packets captured
- **Bytes**: Total data transferred

### Advanced Monitor Also Shows
```json
"google.com": {
  "packets": 245,
  "bytes": 156234,
  "ips_count": 3,
  "ips": ["142.250.185.46", "..."],
  "last_seen": "2024-04-26 14:32:44"
}
```
- **ips_count**: How many different servers accessed
- **ips**: Which servers' IP addresses
- **last_seen**: When last activity was

## ❌ Common Problems & Fixes

| Problem | Fix |
|---------|-----|
| "Permission denied" | Use `sudo` |
| "No such device" | Check interface name with `ifconfig` |
| "libpcap not found" | Run `./build.sh` or `brew install libpcap` |
| No traffic captured | Wrong interface? Try `en0` or `eth0` |
| Program hangs | Might be normal - wait few seconds, then Ctrl+C |

## 🔍 Interface Names by Platform

### macOS
- `en0` - Wi-Fi or main ethernet
- `en1` - Secondary ethernet
- `lo0` - Loopback (local only)
- `utun0` - VPN tunnel

### Linux
- `eth0` - Ethernet
- `wlan0` - Wi-Fi
- `docker0` - Docker
- `lo` - Loopback

### Windows (if using)
- Requires different tool (Npcap)
- Not supported by this program

## 🎯 Use Cases (1-Liner Commands)

```bash
# See what websites accessed
sudo ./network_monitor en0

# Monitor Docker container network
sudo ./network_monitor docker0

# Check DNS requests (might indicate malware)
sudo ./network_monitor_advanced en0
# Then: cat traffic_report_*.json | jq '.packets[] | select(.protocol == "DNS")'

# Find most visited websites
sudo ./network_monitor en0  # Ctrl+C, lihat output

# Log continuously untuk 1 jam
timeout 3600 sudo ./network_monitor en0 | tee $(date +%s).log

# Analyze after stopping
jq '.domains | to_entries | max_by(.value.bytes)' traffic_report_*.json
```

## 🛠️ Customize BPF Filter

Edit source code, ubah line:

```cpp
std::string filter_str = "udp port 53 or tcp port 80 or tcp port 443 or tcp port 8080";
```

**Contoh filters**:
```cpp
"udp port 53"                          // DNS only
"tcp port 443"                         // HTTPS only
"host 8.8.8.8"                       // Specific IP only
"(tcp or udp) and port 53"            // Both TCP & UDP DNS
"dst net 192.168.0.0/16"              // Local network only
"tcp port 80 or tcp port 8080"        // HTTP ports only
```

Compile ulang: `make clean && make`

## 💡 Pro Tips

✅ **Use screen/tmux untuk background monitoring**
```bash
screen -S monitor
sudo ./network_monitor_advanced en0
# Ctrl+A then D to detach
screen -r monitor  # Reattach later
```

✅ **Export dan analyze dengan jq**
```bash
cat traffic_report_*.json | jq '.domains | keys | length'  # Count unique domains
```

✅ **Combine dengan grep untuk filter**
```bash
sudo ./network_monitor en0 | tee output.txt
# Later:
grep "google" output.txt
```

✅ **Monitor specific app**
```bash
# Terminal 1
sudo ./network_monitor en0

# Terminal 2
./my-app

# See what it accessed
```

## 🔐 Security Notes

⚠️ **Local only** - Can't monitor other devices (use different tools)  
⚠️ **Read-only** - Won't modify traffic  
⚠️ **No encryption bypass** - Can't see HTTPS content

## 🚨 Emergency Fixes

**Stuck/hanging?**
- Press `Ctrl+C` to stop
- Results will show

**Wrong interface?**
- Stop (Ctrl+C)
- List interfaces: `ifconfig`
- Run again with correct interface

**Performance issues?**
- Filter more specifically
- Use wlan0 instead of en0 if available
- Run on less busy network
